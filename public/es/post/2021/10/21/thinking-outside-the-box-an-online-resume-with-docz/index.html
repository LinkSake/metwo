Pensando fuera de la caja: un currículum en línea con Docz

por Luis Angel Ortega (jueves, 21 de octubre de 2021) | 625 palabras / 3 minutos

Una de las mejores cualidades de la humanidad es su curiosidad. Es tan humana que en honor a ella nombramos a un rover de Marte “Curiosity” en 2003.

Hablando de curiosidad en el mundo de los desarrolladores, los proyectos de código abierto están a la cabeza, ya que por naturaleza nos invitan a experimentar, modificar y compartir qué más podemos hacer con ellos. Es por eso que propongo pensar un poco fuera de la caja con Docz - una herramienta de documentación de software que es completamente de código abierto.

¿Qué es Docz?

Docz es un proyecto basado en Gatsby que simplifica el proceso de documentar otros proyectos sin preocuparse por la configuración, la velocidad y el soporte.

Proporciona una biblioteca que permite escribir páginas MDX en lugar de archivos HTML o JSX, maneja el enrutamiento y ofrece complementos para todas las demás necesidades que puedas tener. ¡Todo esto resulta en proyectos a los que cualquiera puede contribuir!

Es esta simplicidad y amabilidad lo que hace de Docz una gran opción para algo más que solo documentación.

Creando nuestro proyecto Docz

Antes de poder empezar con Docz, hay algunos requisitos previos que necesitarás tener para crear el proyecto:

  • NodeJS
  • Yarn o npm

Ahora que tienes los requisitos previos, comencemos creando una nueva aplicación React con create-react-app (CRA)

Ve a tu terminal y ejecuta:

npx create-react-app mi-curriculum

Si no tienes CRA instalado, npm te preguntará si está bien instalar create-react-app escribe y para continuar y deja que CRA cree el nuevo proyecto.

Estamos usando el comando npx en lugar de npm ya que queremos ejecutar un paquete, no instalarlo en un proyecto. Puedes leer más sobre esto aquí.

Ahora movámonos al directorio de nuestro proyecto e instalemos Docz.

cd mi-curriculum
yarn add docz # o npm install docz

Luego, eliminemos todo lo que CRA creó para nosotros dentro de la carpeta src, ya que no lo necesitamos.

rm src/*

También, es una buena idea agregar a .gitignore la carpeta .docz ya que solo la necesitamos para el desarrollo.

# .gitignore

# Archivos generados por Docz
.docz/

Vamos a crear un archivo llamado index.mdx y un encabezado hola mundo de la siguiente manera.

<!-- src/index.mdx -->

---
name: Hola Mundo
route: /
---

# ¡Hola mundo desde Docz!

Y es hora de ejecutar nuestro servidor de desarrollo para ver lo que acabamos de crear.

yarn docz dev # o npm run docz dev

Si obtienes un error ERROR #98123 al intentar ejecutar el servidor de desarrollo, simplemente elimina tu yarn.lock o package-json.lock y la carpeta node_modules e instala las dependencias nuevamente.

¡Et voila! Hemos creado con éxito nuestro proyecto Docz.

Ahora es un buen momento para hacer commit de tus archivos!

Tu momento de brillar

Antes de comenzar a agregar más páginas, creemos un archivo de configuración llamado doczrc.js en la raíz de nuestro proyecto. Esto nos ayudará a establecer algunas metaetiquetas fácilmente, pero gestiona toda la configuración de nuestro proyecto.

// doczrc.js

export default {
  title: "Luis Angel Ortega",
  description: "Hola, soy Luis Angel y este es mi currículum hecho con Docz!",
  ignore: ["README.md"]
}

La clave title establecerá el sufijo para nuestra etiqueta title y cambiará el texto en la esquina superior izquierda de nuestro proyecto, ya que es un currículum en línea, recomiendo usar tu nombre en este campo.

Luego, la clave description que agregamos modificará la etiqueta meta description en nuestra página web para mostrarla cuando se busque o comparta en línea.

La última clave le dirá a Docz que ignore algunos archivos y no los muestre en la página web, como es en este caso con el archivo README.
